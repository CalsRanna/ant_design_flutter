## 0. 决策摘要

| 维度 | 决策 |
| --- | --- |
| 演进策略 | 同仓库发 2.0 大版本，v1-legacy tag 冻结 1.x |
| 设计源 | Ant Design v5（推荐锁定 v5.22.x） |

## 1. 架构总览

### 1.1 分层

```
L0  Foundation  颜色/字体/图标常量、色板算法
L1  Theme       AntTheme InheritedWidget + AntThemeData (Seed/Map/Alias Token)
L2  Primitives  AntPortal / AntInteractionDetector / AntOverlayManager
L3  Components  17 个 MVP 组件
L4  Form        AntFormController + AntFormField mixin
L5  App Shell   AntApp / AntConfigProvider
```

**硬约束**：高层可依赖低层，反向禁止；同层之间只能通过公开 API 交互，不许内部互引。

### 1.2 包导出策略

替代旧 1.x 的劫持式导出（旧版 `export 'package:flutter/widgets.dart' hide ...`）：

- `package:ant_design_flutter/ant_design_flutter.dart` 只导出本库符号
- 与 Flutter 原生同名符号一律加 `Ant` 前缀：`AntIcon / AntImage / AntTable / AntSize`
- 用户无需为本库添加别名 import

## 3. Design Token 系统

### 3.1 三层 Token

```dart
// Seed：用户唯一需要配置的源
class AntSeedToken {
  const AntSeedToken({
    this.colorPrimary = const Color(0xFF1677FF),
    this.colorSuccess = const Color(0xFF52C41A),
    this.colorWarning = const Color(0xFFFAAD14),
    this.colorError   = const Color(0xFFFF4D4F),
    this.colorInfo    = const Color(0xFF1677FF),
    this.colorTextBase = const Color(0xFF000000),
    this.colorBgBase   = const Color(0xFFFFFFFF),
    this.fontFamily,
    this.fontSize      = 14,
    this.borderRadius  = 6,
    this.sizeUnit      = 4,
    this.sizeStep      = 4,
    this.wireframe     = false,
    this.motion        = true,
  });
}

// Map：算法从 seed 派生
class AntMapToken {
  final List<Color> colorPrimary;     // primary-1 ~ primary-10
  final List<Color> colorSuccess;
  final List<Color> colorNeutral;     // 灰阶 13 级
  final double controlHeight;
  final double controlHeightLG;
  final double controlHeightSM;
  final BoxShadow shadow1;
  final BoxShadow shadow2;
}

// Alias：组件直接消费的语义层
class AntAliasToken {
  final Color colorBorder;
  final Color colorBgContainer;
  final Color colorBgElevated;
  final Color colorText;
  final Color colorTextSecondary;
  final Color colorTextTertiary;
  final Color colorTextDisabled;
  final Color colorPrimaryHover;
  final Color colorPrimaryActive;
  // ...
}
```

### 3.2 算法接口（扩展点）

```dart
abstract interface class AntThemeAlgorithm {
  AntMapToken mapFromSeed(AntSeedToken seed);
  AntAliasToken aliasFromMap(AntSeedToken seed, AntMapToken map);
}

class DefaultAlgorithm implements AntThemeAlgorithm { ... }
// 预留：DarkAlgorithm、CompactAlgorithm（2.1）
```

### 3.3 色板生成

对齐 [@ant-design/colors](https://github.com/ant-design/ant-design-colors) 的 `generate` 函数：基于 HSV 色彩空间按固定的 hue/saturation/value 偏移生成 10 阶。Dart 端约 200 行实现，单测对拍 JS 版输出。

### 3.4 消费规约

组件**只消费 Alias Token**，不直接读 Seed/Map。这样新增 algorithm 时组件零改动。

```dart
final alias = AntTheme.of(context).alias;
DecoratedBox(
  decoration: BoxDecoration(color: alias.colorPrimaryHover),
  ...
)
```

### 3.5 嵌套覆盖

```dart
AntConfigProvider(
  theme: AntThemeData(seed: AntSeedToken(colorPrimary: Colors.purple)),
  child: SomeSubtree(),
)
```

通过嵌套 `AntTheme` InheritedWidget 实现局部主题覆盖。

## 4. Primitives 基础设施（L2）

### 4.1 AntInteractionDetector

封装 `WidgetStatesController`，吞掉 hover/focus/press/disabled 五种状态的样板代码：

```dart
class AntInteractionDetector extends StatefulWidget {
  const AntInteractionDetector({
    required this.builder,
    this.onTap,
    this.onHover,
    this.enabled = true,
    this.focusable = true,
    this.cursor,
  });

  final Widget Function(BuildContext, Set<WidgetState>) builder;
  final VoidCallback? onTap;
  final ValueChanged<bool>? onHover;
  final bool enabled;
  final bool focusable;
  final MouseCursor? cursor;
}
```

### 4.2 AntPortal

弹层定位，基于 `CompositedTransformTarget` / `CompositedTransformFollower`：

```dart
class AntPortal extends StatefulWidget {
  final Widget child;
  final WidgetBuilder overlayBuilder;
  final AntPlacement placement;       // 12 方位
  final bool visible;
  final VoidCallback? onDismiss;
  final bool autoAdjustOverflow;
  final Offset offset;
}

enum AntPlacement {
  topLeft, top, topRight,
  rightTop, right, rightBottom,
  bottomRight, bottom, bottomLeft,
  leftBottom, left, leftTop,
}
```

**MVP 自动翻转策略**：layout 后一帧用 `RenderBox.localToGlobal` + `MediaQuery.size` 检查越界，若越界则 flip 到对侧（最多翻转一次避免振荡）。

消费者：Tooltip / Select dropdown / DatePicker panel / Menu submenu / Dropdown。

### 4.3 AntOverlayManager

管理无父级锚点的浮层（Message / Notification / Modal / Drawer）。每个 slot 内部维护有序列表，按插入顺序 vertically stack；用 `GlobalKey` + `addPostFrameCallback` 正确测量高度（解决旧版高度测量始终为 null 的 bug）。

```dart
class AntOverlayManager {
  static OverlayEntryHandle show({
    required BuildContext context,
    required WidgetBuilder builder,
    required AntOverlaySlot slot,
  });
  static void dismiss(OverlayEntryHandle handle);
}

enum AntOverlaySlot { message, notification, modal, drawer }
```

## 5. Form 引擎

### 5.1 公开 API

对齐 AntD v5 `Form.useForm()` 的命令式控制能力：

```dart
final formController = AntFormController();

AntForm(
  controller: formController,
  onFinish: (values) => print(values),
  onFinishFailed: (errors) => ...,
  child: Column(children: [
    AntFormItem(
      name: 'username',
      label: Text('用户名'),
      rules: [
        AntValidators.required('请输入用户名'),
        AntValidators.min(3, '至少 3 个字符'),
      ],
      child: AntInput(),
    ),
  ]),
)

// 命令式
formController.getFieldValue('username');
formController.setFieldsValue({'username': 'alice'});
formController.validateFields().then(...);
formController.resetFields();
formController.submit();

// 响应式（替代 rxdart）
formController.watch('username');   // 返回 ValueListenable<T?>
```

### 5.2 内部实现（零第三方依赖）

```dart
class AntFormController extends ChangeNotifier {
  final Map<String, _FieldState> _fields = {};

  T? getFieldValue<T>(String name) => _fields[name]?.value as T?;

  void setFieldsValue(Map<String, dynamic> values) {
    values.forEach((k, v) => _fields[k]?.setValue(v));
    notifyListeners();
  }

  Future<Map<String, dynamic>> validateFields() async {
    final results = await Future.wait(
      _fields.values.map((f) => f.validate()),
    );
    if (results.any((r) => r != null)) {
      throw AntFormValidationError(_collectErrors());
    }
    return _collectValues();
  }

  void resetFields([List<String>? names]) { ... }

  ValueListenable<T?> watch<T>(String name)
      => _fields[name]!.listenable as ValueListenable<T?>;
}

class _FieldState {
  _FieldState({required this.name, required this.rules});
  final String name;
  final List<AntValidator> rules;
  final ValueNotifier<dynamic> listenable = ValueNotifier(null);
  String? _error;

  Future<String?> validate() async {
    for (final rule in rules) {
      final err = await rule.validate(listenable.value);
      if (err != null) { _error = err; return err; }
    }
    _error = null;
    return null;
  }
}
```

### 5.3 校验器

```dart
abstract class AntValidator {
  Future<String?> validate(dynamic value);   // null=ok，string=error
}

class AntValidators {
  static AntValidator required(String msg);
  static AntValidator min(int n, String msg);
  static AntValidator max(int n, String msg);
  static AntValidator email([String? msg]);
  static AntValidator pattern(RegExp p, String msg);
  static AntValidator custom(Future<String?> Function(dynamic) fn);
}
```

### 5.4 与组件解耦

```dart
mixin AntFormField<T> on StatefulWidget {
  T? get value;
  ValueChanged<T>? get onChanged;
}
```

任何实现该 mixin 的 widget（包括用户自定义）都能被 Form 管理。

## 6. 组件实现策略

### 6.1 MVP 组件清单（按依赖顺序）

```
Round 1  原子层
  ① AntIcon       - 仅依赖 IconData，使用本库自带 AntIcons 字体
  ② AntTypography - Title / Text / Paragraph / Link

Round 2  基础表单控件（依赖 InteractionDetector）
  ③ AntButton
  ④ AntInput
  ⑤ AntCheckbox + CheckboxGroup
  ⑥ AntRadio + RadioGroup
  ⑦ AntSwitch
  ⑧ AntTag

Round 3  反馈类（依赖 OverlayManager）
  ⑨ AntTooltip      （依赖 Portal）
  ⑩ AntMessage
  ⑪ AntNotification
  ⑫ AntModal

Round 4  复合
  ⑬ AntSelect       （Portal + Input 视觉 + Checkbox）
  ⑭ AntDatePicker   （Portal + Input + 日历网格）
  ⑮ AntTabs
  ⑯ AntMenu         （含 SubMenu）
  ⑰ AntTable        （MVP 不含分页与虚拟滚动）

Form（独立并行）
  AntForm / AntFormItem / AntFormController / Validators
```

### 6.2 组件交付定义（DoD）

```
□ 对齐 AntD v5 文档页的 props（按 80/20 原则裁剪，明确声明实现子集）
□ 所有声明的 props 真实生效（不允许装饰性 prop）
□ 至少 1 个 widget test（核心交互路径）
□ widgetbook ≥ 3 个 story（default + 主要变体 + 边界态）
□ 公开 API 有 dartdoc 注释
□ 无 analyzer 警告
□ 在 gallery/ 子项目添加对应 widgetbook story
```

**Golden test 不在 MVP DoD 内**（推迟到 2.1）。

**"prop 真实生效"的判定**：每个 prop 必须有至少一处可见效果（视觉变化 / 行为变化），且能被 widget test 验证。"装饰性 prop" 指字段定义了但 build 函数从不读取的情况，这是 1.x 最普遍的反模式（参见 1.x `AntInput` 的 `onChange/value/maxLength` 等），2.0 严禁出现。

### 6.3 Prop 裁剪原则

- **必做**：影响视觉/交互的主 prop（type / size / disabled / loading / onClick）
- **可做**：对齐 AntD 语义且实现简单的（block / ghost / danger）
- **推迟到 2.1**：边缘 / 高级功能（iconPosition / autoInsertSpaceInButton）
- **永不做**：AntD 自己已 deprecated 的 API

每个组件在自己的 README 段落里列"实现 prop / 推迟 prop"两栏对照表。

### 10.1 Phase 检查点

每完成一个 Phase 做一次 retrospective：
- 实际工时 vs 估算
- 哪些组件比想象难
- 是否需要砍 prop / 缩范围保持节奏

如果 3 个 Phase 后实际进度落后估算 30% 以上，则砍 Table 或 DatePicker 到 2.1。

### 10.2 反完美主义原则

- 每个 Phase 结束打 tag 并发 pub.dev
- dev 版可以丑、可以缺东西，但能跑能用
- 困难章节（DatePicker / Table）允许超 50% 时间，否则全盘节奏崩

## 11. 风险清单与规避
